Rekomendacja — optymalizacja budżetów wg priorytetów

Aby nie przekraczać dotychczasowego budżetu (~210 zł/dzień), warto zróżnicować stawkę na grupę w zależności od priorytetu. Pilne wakaty powinny mieć największy budżet — mniej ważne priorytety można lekko obniżyć, zachowując pełną kontrolę (brak „losowego" podziału Meta).

Problem ze strukturą PRZED

  • Jeden budżet na całą grupę — Meta sama decydowała, która reklama w grupie dostaje więcej środków
  • Nierówny podział — np. P1 miało 8 reklam przy 50 zł → ~6,25 zł/reklamę; P2 Masażyści 3 reklamy przy 30 zł → 10 zł/reklamę
  • Brak kontroli nad tym, która reklama rzeczywiście „wychodzi" do ludzi
  • Priorytety w ramach jednej kampanii — trudno analizować wyniki per wakacja

Zalety struktury PO

  • Każda grupa = 1 reklama → pełna kontrola budżetu per ogłoszenie
  • Równy start — każda reklama dostaje tyle samo, wyniki porównywalne
  • Priorytety = osobne kampanie → raportowanie per priorytet proste i czyste
  • Skalowanie — łatwo podnieść budżet tylko tym, które działają

🔗 Jak podłączyć Claude do Meta Ads API?

Automatyczne raporty z wpisywaniem nazwy reklamy i podglądem wyników w czasie rzeczywistym

01

Meta Business → Token API

Wejdź do Meta for Developers, utwórz aplikację typu „Business", wygeneruj Access Token z uprawnieniami ads_read i ads_management. Token wklej do pliku .env projektu.

META_ACCESS_TOKEN=...
02

Mini-strona do wpisywania nazwy

Prosta strona HTML z polem tekstowym (fragment nazwy reklamy). Po kliknięciu „Szukaj" zapytanie idzie do backendu (np. Python/Flask), który odpytuje Meta API po ad_name i zwraca wydatki + konwersje.

GET /ads?q=Masażyści
03

Wyniki na ekranie

Odpowiedź pokazuje: spend (wydane złotówki), impressions, clicks, konwersje i koszt/wynik. Odświeżenie = nowe dane z Meta API. Koszt wdrożenia: niski (tylko hosting backendu ~0–20 zł/mies.).
